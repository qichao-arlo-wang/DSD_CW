\section{Task 7a: CORDIC Analysis (10\%)}
% =============================================================

\subsection{Reference Model and Statistical Method}
Task 7a uses a MATLAB Monte Carlo model with:
\begin{itemize}
    \item random input $\theta \sim U[-1,1]$, quantised to \texttt{single};
    \item reference output $y_{ref}=\cos(\theta)$ under MATLAB single precision;
    \item CORDIC estimate $y_{cordic}$ using fixed-point internal arithmetic;
    \item error sample $e_k=(y_{cordic,k}-y_{ref,k})^2$;
    \item MSE estimate $\hat{\mu}_e=\frac{1}{N}\sum_{k=1}^{N}e_k$ and 95\% CI
    $\hat{\mu}_e \pm 1.96\cdot s_e/\sqrt{N}$.
\end{itemize}
For all reported Task 7a results, we used $N=50{,}000$ samples and seed $=7$.

\subsection{Design Space Exploration}
We swept both CORDIC parameters required by the specification:
\begin{itemize}
    \item iterations: all integers $N_{iter}\in\{12,13,\dots,24\}$;
    \item fixed-point precision: all integer FRAC values from 20 to 34.
\end{itemize}
The total fixed-point width is set as:
\[
W = \mathrm{FRAC} + 2.
\]
The +2 integer/sign bits cover the CORDIC dynamic range for $\theta\in[-1,1]$ and $\cos(\theta)\in[-1,1]$.
The acceptance criterion is the coursework requirement:
\[
\text{CI}_{95,\text{upper}}(\text{MSE}) < 2.4\times10^{-11}.
\]

\subsection{Analytical Error Estimate ($2^{-i}$)}
For rotation-mode CORDIC, the residual angle after $i$ iterations is bounded by a term proportional to $2^{-i}$.
Using $|\Delta\cos(\theta)|\approx |\sin(\theta)\Delta\theta|\le|\Delta\theta|$, the absolute cosine error also follows an $O(2^{-i})$ trend; therefore squared error follows approximately $O(2^{-2i})$.
This estimate is not used as the final proof, but it is effective to prune the search space before Monte Carlo.

\subsection{Selected Configuration and Justification}
After including odd values, the first passing point in lexicographic order
(minimum iterations, then minimum FRAC) is:
\[
(N_{iter},W,\text{FRAC})=(17,23,21).
\]
Measured statistics for this minimum-passing point are:
\begin{itemize}
    \item MSE $=2.254\times10^{-11}$,
    \item 95\% CI $=[2.226, 2.282]\times10^{-11}$,
    \item CI upper bound margin to spec: $2.4\times10^{-11}/2.282\times10^{-11}\approx 1.05\times$.
\end{itemize}
For implementation in Task 7b/7c we keep $(18,24,22)$ as a conservative choice:
with \texttt{ITER\_PER\_CYCLE}=3, both 17 and 18 iterations map to the same CORDIC cycle count (6 cycles), while $(18,24,22)$ has significantly larger numerical margin.

\subsection{Impact on Accuracy and Latency}
Observed impact from the full sweep:
\begin{itemize}
    \item For FRAC=20, the threshold is crossed at $N_{iter}=18$ (17 fails, 18 passes).
    \item For FRAC$\ge$21, $N_{iter}=17$ already passes.
    \item Increasing FRAC beyond the threshold region keeps reducing error, but with diminishing practical benefit.
    \item Latency grows with iteration count (for fixed iterations-per-cycle), while larger FRAC increases datapath cost; therefore minimum-passing and implementation-conservative points are both reported.
\end{itemize}
